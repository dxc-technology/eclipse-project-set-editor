Projekt Set

Konzept

Ein Projekt Set hält spezifische Informationen bereit, um ein Bündel von Projekten eines Team-Repositories im Arbeitsbereich wieder herzustellen. Nur Projekte auf die gemeinsam zugegriffen wird, können Teil eines Projekt Sets sein.

Zum Beispiel können Projekt Sets benutzt werden, um Projekte für eine Veröffentlichung zu gruppieren oder um sie einer anderen Person zukommen zu lassen.

Projekt Sets werden in XML-Dateien mit der Datei-Erweiterung .psf gespeichert. In den Eclipse Ansichten werden die Dateien mit dem Symbol dargestellt.

Das Format einer Projekt Set Datei, welches von diesem Plugin benutzt wird, ist dasselbe, welches in Eclipse für das Exportieren oder Importieren in Team Projekt Sets verwendet wird.

Die Implementierung dieses Plugins unterstützt aktuell CVS und Git als Team-Repositories. Andere Team-Repository Anbieter können Erweiterungen zur Unterstüzung dieses Projekt Set Plugins erstellen.

Bevorzugte Projektverzeichnisse

Im Projekt Set Editor kann man für jedes Projekt ein bevorzugtes lokales Projektverzeichnis angeben, in das das Projekt beim ersten Abrufen aus dem Team-Repository gespeichert wird. Wird kein bevorzugtes Verzeichnis angegeben, so wird das Projekt standardmäßig in das Eclipse Workspace Verzeichnis gespeichert. Ist das Projekt bereits in ein anderes Verzeichnis abgerufen worden, so wird dieses Verzeichnis verwendet.

Die Informationen über die bevorzugten Projektverzeichnisse werden in einer zusätzlichen Datei gespeichert, die den gleichen Namen wie die Projekt Set Datei hat, jedoch mit dem Dateizusatz .ppl. Die .ppl-Datei wird mit dem Icon angezeigt und wird erzeugt sobald der Benutzer im Projekt Set Editor das erste mal ein bevorzugtes Projektverzeichnis angibt und das Projekt Set speichert.

Bei der Angabe der bevorzugten Projektverzeichnisse können Platzhalter verwendet werden. Die Syntax für die Verwendung ist ${Variablenname}. Zum Beispiel kann man "${myroot}/subfolder" als bevorzugtes Verzeichnis angeben und jeder Benutzer kann sein eigenes "myroot"-Verzeichnis definieren. Jeder verwendeten Variable muss in der Benutzervorgabenseite für Projekt Sets ein Wert zugewiesen werden. Die Pfadvariable ${workspace} ist immer implizit definiert und enthält den Pfad zum Eclipse Workspace Verzeichnis. Projekt Sets mit undefinierten Variablen können nicht geladen werden.

Achtung!

Das angegebene bevorzugte Projektverzeichnis wird beim Abrufen des Projektes zuvor gelöscht. Geben Sie hier bitte immer den vollständigen Pfad des Verzeichnisses an, in das die Projektdateien abgerufen werden sollen. Wenn Sie z.B. C:\ als Pfad angeben, wird ihre komplette Festplatte gelöscht!

Hinweis

Beim Abbrufen eines Projektes mit einem bevorzugten Projektverzeichnis erscheint immer die Meldung "Das Projekt ist bereits vorhanden. Soll es überschrieben werden?", auch wenn es zuvor nicht vorhanden war. Aus technischen Gründen läßt sich diese Meldung zur Zeit nicht verhindern. Bitte bestätigen Sie die Meldung mit "Ja".

Erstellen einer Projekt Set Datei

Um eine Projekt Set Datei zu erstellen gehen Sie über das Menü "Datei > Neu > Andere ..." und wählen dann die Kategorie "Projekt Set" und den Typ "Projekt Set".

Der Wizard für die Erstellung von neuen Projekt Sets hat zwei Seiten.

Auf der ersten Seiten können Sie den Namen der Projekt Set Datei spezifizieren und den Zielordner eingeben.

Auf der zweiten Seite wählen Sie die Projekte aus, die in das Projekt Set kommen. Nur gemeinsam genutzte Projekte im Arbeitsbereich können hinzugefügt werden. Sie werden mit der Kennzeichnung (Version/Branch) hinzugefügt, mit der das Projekt im Arbeitsbereich geladen ist. Der Dialog "Inhalt des Projekt Sets" besitzt einige Knöpfe, um die Auswahl zu vereinfachen (z.B. Wähle alle Projekte vom Arbeitsset). Wenn die Checkbox "Selektiere referenzierte" aktiviert ist, werden die nachfolgend selektierten Projekte auch ihre referenzierten Projekte selektieren. Das funktioniert auch rekursiv. Die Checkbox "Selektiere referenzierte" wendet ihre Funktion nicht auf bereits selektierte Projekte an. Das Abwählen eines Projektes, wählt nicht automatisch seine referenzierten Projekte ab.

Beachten Sie: Um eine Projekt Set Datei zu erstellen, können Sie auch das Menü "Datei > Export" nutzen und darin "Team Projekt Set" wählen.

Eine Projekt Set Datei bearbeiten

Um eine Projekt Set Datei zu bearbeiten

  • führen Sie einen Doppelklick auf die Projekt Set Datei in einer Eclipse-Sicht aus oder

  • wählen Sie "Öffnen" oder "Öffnen mit > Projekt Set Editor"

Ein Editor für das Projekt Set wird dann geöffnet. Wie dieser Editor funktioniert, wird im Dokument Projekt Set Editor beschrieben.

Laden/Ersetzen von Projekten durch ein Projekt Set

Die Projekte, auf die in einer Projekt Set Datei verwiesen wird, können zum Arbeitsbereich hinzugefügt werden, indem

  • mit einem Projekt Set Editor die Projekt Set Datei geöffnet wird und Sie dann die Funktionalität des Editors nutzen oder

  • das Menü "Lade/Ersetze Projekte" im Kontextmenü der Projekt Set Datei in einer Eclipse-Sicht benutzt wird oder

  • das Menü "Lade/Ersetze Projekte rekursiv" im Kontextmenü der Projekt Set Datei in einer Eclipse-Sicht benutzt wird (die Bedeutung des rekursiven Ladens ist in Projekt Set Projekt erklärt) oder

  • Sie in das Menü "Datei > Import... " gehen und "Team Projekt Set" wählen.

Wenn ein Projekt bereits in den Arbeitsbereich geladen ist, werden Sie gefragt, ob der Inhalt überschrieben werden soll.

Bemerkungen

Angaben in Projekt Sets über den Ort von Repositories unterscheiden zwischen Servernamen und IP Adressen. Das bedeutet, dass es zu Problemen führt, wenn ein Benutzer den Servername verwendet und ein anderer die IP Adresse des Servers (oder einen anderen Namen für den Server). So wird dann z.B. ein Projekt für einen Benutzer als "andere Version geladen" angezeigt, obwohl er die gleiche Version vom physikalisch identischen Server geladen hat. Es empfiehlt sich die Servernamen anstatt der IP Adressen zu verwenden.

Um Probleme beim Verschieben eines Repositories zu vermeiden, empfehlen wir für jedes Repository einen eigenen symbolischen Servernamen zu verwenden. Jeder Benutzer ordnet in seiner hosts Datei diesen symbolischen Namen die realen Servernamen oder die IP Adressen zu. Unter Windows NT/2000 liegt diese Datei im Verzeichnis C:\WINNT\system32\drivers\etc.
