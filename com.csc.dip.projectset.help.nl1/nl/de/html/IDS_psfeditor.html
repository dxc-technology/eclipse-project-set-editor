Projekt Set Editor

Übersicht Editor

Der Projekt Set Editor (PSF Editor) wird benutzt um eine Projekt Set Datei zu editieren. Er hat drei Bereiche:

  • Die Tabelle im obersten Bereich zeigt den Inhalt des Projekt Sets.

  • Der untere Bereich zeigt die Details des Projekt Set Eintrages, der in der Tabelle ausgewählt ist.

  • Auf der rechten Seite befinden sich die Knöpfe zum Bearbeiten des Projekt Sets.

Beschreibung der Projekt Set Inhaltstabelle

Die Tabelle zeigt die Einträge des Projekt Sets. Sie hat folgende Spalten:

  • Name: Icon, Name und Repositoryinformationen des gerade geladenen Projektes. Nur der Name, falls das Projekt nicht geladen ist.

  • Kennzeichnung: Die Kennzeichnung des Projektes im Projekt Set. Für CVS als Team Provider ist dies "HEAD" oder der Versions- oder Verzweigungsname.

  • Verzeichnisprobleme: Zeigt Probleme, die sich bei der Angabe des bevorzugten lokalen Verzeichnisses ergeben. Mögliche Probleme sind:

    • Das spezifizierte bevorzugte lokale Verzeichnis enthält undefinierte Pfadvariablen

    • Das Projekt ist nicht in das bevorzugte lokale Verzeichnis geladen

    • Das bevorzugte lokale Verzeichnis ist identisch mit (oder enthalten in) einem bevorzugten Verzeichnis eines anderen Projektes

    • Verschiedene bevorzugte Verzeichnisse wurden für ein Projekt angegeben. (Nur möglich wenn Projekt Set Projekte benutzt werden)

    In der Spalte können folgende Symbole angezeigt werden:

    • Der Eintrag hat ein Pfadproblem

    • Ein Untereintrag hat ein Pfadproblem

  • Kennz. Konflikte: Zeigt die Konflikte , wennauf ein Projekt mit verschiedenen Kennzeichnungen innerhalb eines Projekt Sets referenziert wird (nur möglich wenn Projekt Set Projekte benutzt werden). Die Spalte zeigt folgendes Symbol an:

    • wenn der Eintrag einen Kennzeichnungskonflikt aufweist.

    • wenn ein Unterpunkt (Eintrag) einen Kennzeichnungskonflikt aufweist

  • Status: Zeigt den Status des Projekteintrages an. Die möglichen Stati sind:

    • Geladen = das Projekt wurde mit der Kennzeichnung, die das Projekt Set vorgegeben hat, geladen.

    • Nicht geladen = das Projekt ist nicht in den Arbeitsbereich geladen.

    • Unterschiedlich geladen = das Projekt ist mit einer anderen Kennzeichnung als der durch das Projekt Set vorgegebenen geladen.

    • Nicht gemeinsam genutzt = das Projekt im Arbeitsbereich wird nicht gemeinsam genutzt.

  • Provider: Der Team Provider des aktuellen Eintrags. Um die Übersichtlichkeit zu verbessern, hat diese Spalte die Breite null, die bei Bedarf aber verändert werden kann.

  • Server Ort: Der Server Ort des aktuellen Eintrags. Um die Übersichtlichkeit zu verbessern, hat diese Spalte die Breite null, die bei Bedarf aber verändert werden kann.

Das Kontextmenü der Tabelle besitzt dieselben Aktionsmöglichkeiten wie auf der rechten Seite des Editors. Wenn alle ausgewählten Projekte geladen sind kommen zusätzlich eigene Projektaktionen (z.B. vergleiche mit, ersetze durch,...) hinzu. Die zusätzlichen Aktionen werden auf alle geladenen Versionen der Projekte angewendet!

Die Einträge können nach dem Inhalt einer Spalte sortiert werden. Dazu klickt man auf die Überschrift der entsprechenden Spalte. Ein zweiter Klick ändert die Sortierung von aufsteigend nach absteigend. Bei einem dritten Klick werden die Einträge nicht mehr sortiert.

Beschreibung des Detail-Bereiches

Der Bereich unterhalb der Tabelle zeigt die Details des ausgewählten Eintrages.

Oben in diesem Bereich wird der Name des ausgewählten Eintrages angezeigt.

In der nächsten Zeile befindet sich ein Eingabefeld für das bevorzugte lokale Verzeichnis. Ein leerer Eintrag bedeutet, dass das Standardverzeichnis innerhalb des Workspace-Verzeichnisses verwendet wird. Der Eintrag kann nur bearbeitet werden, wenn der Team-Provider ein bevorzugtes lokales Verzeichnis anbietet, und ist nur für Wurzeleinträge editierbar, d.h. nicht für Untereinträge. Ein Untereintrag muss in dem Projekt Set bearbeitet werden, das ihn direkt enthält. Die Knöpfe neben dem Eingabefeld habe folgende Funktionen:

  • Durchsuchen: Öffnet einen Dateidialog zur Auswahl eine lokalen Verzeichnisses

  • Aktuell: Setzt das bevorzugte lokale Verzeichnis auf das Verzeichnis, in dem das Projekt aktuell gespeichert ist

  • Variable: Öffnet einen Dialog zur Auswahl einer Pfadvariablen, die dann in dem Eingabefeld eingefügt wird. (Die Variablen werden in den Projekt Set Benutzervorgaben definiert)

Unterhalb des Eingabefeldes werden die Kennzeichnungskonflikte und die Probleme bezüglich des bevorzugten lokalen Verzeichnisses angezeigt.

Die beiden Bereiche im unteren Teil des Editors zeigen noch mehr Details über den ausgewählten Eintrag und das geladene Projekt. Die Bereiche können durch Klicken der blauen Dreiecke neben den Bereichsüberschriften auf- und zugeklappt werden. Das bevorzugte lokale Verzeichnis wird hier mit aufgelösten Pfadvariablen gezeigt. Unterschiede zwischen dem Eintrag und dem geladenen Projekt sind rot markiert.

Beschreibung der Knöpfe

Die Knöpfe für die möglichen Aktionen auf der rechten Seite des Editors sind:

  • Lade/Ersetze ausgewählte: Lädt oder ersetzt (wenn vorhanden) die ausgewählten Projekte durch die in dem Projekt Set angegebenen Kennzeichnung.

  • Lade/Ersetze alle: Lädt oder ersetzt (wenn vorhanden) alle Projekte des Projekt Sets durch die Kennzeichnung, die in dem Projekt Set angegeben ist.

  • Hinzufügen: Öffnet einen Dialog für die Auswahl der Projekte, die zu dem Projekt Set hinzugefügt werden sollen. Nur gemeinsam genutzte Projekte, die noch nicht im Projekt Set sind, werden aufgelistet.

  • Entfernen: Entfernt alle ausgewählten Einträge vom Projekt Set.

  • Setze ausgw. zu geladener Kennz.: Setzt die Kennzeichnungder ausgewählten Einträge zu der Kennzeichnung, welches das Projekt in Arbeitsbereich hat.

  • Alle auswählen: Wählt alle Einträge im Projekt Set aus.

  • Geladene auswählen: Wählt alle Einträge im Projekt Set aus, die im Workspace geladen sind, auch die, welche in einer anderen Version geladen sind.

  • Ungeladene auswählen: Wählt alle Einträge im Projekt Set aus, deren Projekt nicht im Workspace geladen ist.

  • Öffne eingeschlossene PS Datei: Öffnet einen neuen PSF Editor für die Projekt Set Datei des ausgewählten Projekt Set Projekts.

Die Bearbeitungsknöpfe zum Hinzufügen, Entfernen oder Ändern der Kennzeichnung sind ausgeschaltet, wenn Untereinträge ausgewählt sind. Diese Einträge müssen im PSF Editor für ihr übergeordnetes Projekt geändert werden. Die "Lade/Ersetze" Knöpfe sind ausgeschaltet, wenn ein Eintrag und gleichzeitig einer seiner übergeordneten Einträge ausgewählt ist.

Die Checkbox "Lade/Ersetze rekursiv" bestimmt, dass die Projekte rekursiv geladen werden, wenn einer der beiden "Lade/Ersetze" Knöpfen gedrückt wird. Der Vorgabestatus dieser Checkbox kann in den Benutzervorgaben für das Projekt Set festgelegt werden.

Speichern des Projekt Sets

Wenn Sie <CTRL> und "s" drücken oder "Speichere Projekt Set" aus dem Kontextmenü des Editors wählen, wird das Projekt Set gespeichert. Auch wenn Sie den Editor schließen, werden Sie gefragt, ob ihre Änderungen in der Projekt Set Datei gespeichert werden sollen.
